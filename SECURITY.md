# Security Policy

## Reporting a vulnerability

If you discover a security vulnerability in any of the Azure Functions Python DX Toolkit packages, please report it responsibly.

**Do not open a public issue.**

Instead, please email the maintainer directly or use GitHub's private vulnerability reporting feature.

## Supported versions

Only the latest version of each package is supported with security updates.

## Scope

This hub repository does not contain executable code. Security issues are most likely to occur in the individual packages:

- [azure-functions-openapi-python](https://github.com/yeongseon/azure-functions-openapi-python)
- [azure-functions-doctor-python](https://github.com/yeongseon/azure-functions-doctor-python)
- [azure-functions-validation-python](https://github.com/yeongseon/azure-functions-validation-python)
- [azure-functions-logging-python](https://github.com/yeongseon/azure-functions-logging-python)
- [azure-functions-scaffold-python](https://github.com/yeongseon/azure-functions-scaffold-python)
- [azure-functions-cookbook-python](https://github.com/yeongseon/azure-functions-cookbook-python)
- [azure-functions-db-python](https://github.com/yeongseon/azure-functions-db-python)
- [azure-functions-langgraph-python](https://github.com/yeongseon/azure-functions-langgraph-python)

Please report vulnerabilities to the specific package repository.
